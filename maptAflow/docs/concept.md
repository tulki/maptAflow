# maptAflow — Concept

## Product Definition

maptAflow — это пространственная система управления задачами.

Система помогает пользователю переходить от стратегического мышления к выполнению задач без потери контекста.

Основная проблема, которую решает система:

разрыв между

Planning → Structuring → Execution

maptAflow оптимизирует именно **исполнение задач**.

---

# Core Idea

Основная метафора системы — **пространственная карта задач**.

Каждая задача представлена узлом.

Узел может:

- содержать информацию о задаче
- иметь подзадачи
- быть связан с другими задачами
- раскрываться в отдельное пространство

---

# Graph Model

Структура системы представляет собой направленный граф задач.

Основные правила:

- у узла может быть только один родитель
- у узла может быть любое количество детей
- циклы запрещены

Таким образом структура представляет собой набор деревьев задач.

---

# Infinite Canvas

Основным пространством работы является **бесконечный холст**.

Пользователь может:

- перемещаться по холсту
- создавать новые узлы
- связывать узлы
- формировать структуру задач

---

# Three-Level Cognitive Model

Система разделяет мышление на три уровня.

---

## Global Level — Strategy

Глобальная карта проектов.

Характеристики:

- бесконечный холст
- граф связанных задач
- крупные узлы

---

## Node Level — Tactics

Тактический уровень.

Каждый узел может раскрывать собственное дерево подзадач.

---

## Local Level — Execution

Минимальный интерфейс для выполнения задач.

Цель уровня — уменьшить трение между задачами и сохранить состояние потока.

---

# Node Concept

Узел является основным элементом системы.

Узел может представлять:

- задачу
- проект
- группу задач
- контейнер информации

---

# UX Principles

Система строится вокруг нескольких принципов.

Flow over features.

Минимальное трение.

Быстрая реакция интерфейса.

Минимальное количество диалогов и модальных окон.

Фокус на продолжении работы, а не на контроле пользователя.
